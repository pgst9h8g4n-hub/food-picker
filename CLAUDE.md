# CLAUDE.md — 美食收藏与随机选择

## 项目工作规范

### 交流原则
- 与用户的所有交流**只用中文**
- **简洁直接，不绕弯子**
- 不清楚需求时**主动确认**，不猜测执行

### 目录结构

| 路径 | 说明 |
|------|------|
| `Docs/需求说明.md` | 用户需求整理，功能列表，优先级 |
| `Docs/技术规范.md` | 技术选型、架构设计、编码规范 |
| `Docs/执行计划.md` | 分步开发计划，每步的具体任务 |
| `开发日志/` | 按日期追加记录每日完成事项和待办事项 |

### 开发日志格式
文件名：`开发日志/YYYY-MM-DD.md`
```
## 完成事项
- ...

## 待办事项
- ...
```

### 开发流程
1. **先整理需求**：把需求梳理清楚，不确定的地方逐一确认
2. **分步规划**：梳理开发步骤，每次只做适量工作，保证项目稳定推进
3. **逐步执行**：不要一口气做太多，按步骤来，每步完成后记录到开发日志
4. **避免返工**：每步完成后确认功能正常再继续下一步
