# 美食收藏与随机选择 — 技术规范

## 技术栈

| 层次 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 构建 | Vite | 5.x | 快速构建 |
| 语言 | TypeScript | 5.x | 类型安全 |
| 框架 | React | 18.x | 组件化 |
| 路由 | TanStack Router | latest | 类型安全路由 |
| 样式 | TailwindCSS | 4.x | 移动端优先 |
| UI | shadcn/ui | latest | 轻量可定制 |
| 状态 | Zustand | latest | 客户端状态 |
| 服务端状态 | TanStack Query | latest | 缓存 + 自动刷新 |
| 表单 | React Hook Form + Zod | latest | 类型安全表单 |
| 后端 | Supabase | latest | PostgreSQL + Auth + RLS |
| 图标 | Lucide React | latest | 简洁图标 |

## 编码规范

1. **组件命名**：PascalCase，文件名与组件名一致
2. **函数命名**：camelCase，动词开头（useFoods, pickRandom）
3. **常量命名**：UPPER_SNAKE_CASE
4. **TypeScript**：严格模式，禁止 `any`
5. **注释**：关键逻辑加注释，中文注释
6. **错误处理**：try-catch + 用户友好的 toast 提示

## Supabase 配置

### RLS 策略
- 启用所有表的 Row Level Security
- 简单策略：用户只能查看自己的数据
- 如需共享：后续可扩展为互相可见
