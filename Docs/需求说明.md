# 美食收藏与随机选择 — 需求说明

## 项目概述

一个供两个人使用的 Web 应用，用于收藏从各种社交平台（小红书、抖音、美团等）发现的美食，并在需要时通过多种条件随机筛选一道菜。

## 核心功能

### P0 — 必须实现
1. **用户认证**：邮箱密码登录，仅两人使用
2. **美食录入**：手动表单录入美食信息（店名、菜品、城市、标签、价格、评分、来源链接等）
3. **美食列表**：卡片式展示，移动端友好
4. **随机挑选**：
   - 按城市筛选（人在成都就从成都的收藏里随机）
   - 按标签筛选（川菜/日料/麻辣等）
   - 按价格区间筛选
   - 排除已吃过的
   - 权重随机：评分高的菜被选中的概率更大（指数权重 2^rating）
5. **数据持久化**：Supabase 云端存储，手机电脑同步

### P1 — 后续迭代
6. **链接自动解析**：粘贴小红书/抖音/美团链接，自动提取店名、菜品等信息
7. **随机历史记录**：记录每次随机结果

## 用户画像
- 用户 A（创建者）：美食爱好者，经常刷小红书/抖音/美团发现好吃的
- 用户 B（朋友）：另一位美食爱好者
- 两人共享同一个收藏列表（互相可见）

## 使用场景
- 日常：刷到好吃的美食，随手记下来
- 用餐时：打开手机，选个城市/标签，随机出一道菜

## 数据模型

### foods 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 创建者（auth.users.id） |
| name | TEXT | 店名/菜品名 |
| city | TEXT | 城市 |
| region | TEXT | 区域/商圈 |
| tags | TEXT[] | 标签数组 |
| price | INTEGER | 价格（元） |
| rating | INTEGER | 评分 1-5 |
| source | TEXT | 来源平台 |
| source_url | TEXT | 原始链接 |
| notes | TEXT | 备注 |
| is_eaten | BOOLEAN | 是否已吃过 |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |
